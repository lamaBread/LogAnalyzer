# 로그 분석 시스템 (LogAnalyzer)

이 시스템은 웹 서버 로그 파일을 분석하여 잠재적인 공격이나 악의적인 활동을 탐지하는 도구입니다.

## 시스템 구성 요소

본 시스템은 다음 3개의 PHP 파일로 구성되어 있습니다:

### 1. `__functions.php`
기본 유틸리티 함수를 제공하는 파일입니다:
- `readLogFileToArray()`: 로그 파일을 읽어 배열로 변환합니다.
- `groupLogsByIP()`: 로그를 IP 주소별로 그룹화합니다.
- `groupLogsByStatusCode()`: 로그를 HTTP 상태 코드별로 그룹화합니다.
- `groupLogsByStatusCodeRange()`: 로그를 HTTP 상태 코드 범위(100, 200, 300, 400, 500)별로 그룹화합니다.

### 2. `__regex_match_function.php`
정규 표현식을 사용한 공격 패턴 탐지 기능을 제공합니다:
- `readAttackRegexPatterns()`: CSV 파일에서 공격 패턴 정규식을 읽어옵니다.
- `formatRegexPattern()`: 정규식 패턴을 적절한 형식으로 포맷팅합니다.
- `isTooGeneralPattern()`: 너무 일반적인 패턴을 필터링합니다.
- `extractUrlFromLog()`: 로그에서 URL, 메소드, 헤더, 본문 등을 추출합니다.
- `isLogSuspicious()`: 로그가 공격 패턴과 일치하는지 확인합니다.
- `isLogSuspiciousFullScan()`: 모든 패턴을 직접 적용하여 로그를 검사합니다.
- `analyzeLogsForAttacks()`: IP별로 그룹화된 로그를 분석하여 의심 점수를 산출합니다.

### 3. `evaluate_suspicion_for_each_IP.php`
API 엔드포인트를 제공하는 파일로, 클라이언트의 요청을 처리합니다:
- POST 요청을 받아 지정된 로그 파일을 분석합니다.
- IP 주소별로 의심 점수를 계산하고 결과를 JSON 형식으로 반환합니다.
- 필요한 오류 처리 및 HTTP 응답 코드를 관리합니다.

## 작동 방식

1. 클라이언트가 로그 파일 경로를 포함한 POST 요청을 `evaluate_suspicion_for_each_IP.php`에 전송합니다.
2. 시스템은 `__functions.php`의 함수를 사용하여 로그 파일을 읽고 IP별로 그룹화합니다.
3. `__regex_match_function.php`의 함수들이 각 로그 항목을 분석하여 공격 패턴과 일치하는지 확인합니다.
4. 각 IP 주소에 대한 의심 점수가 계산되고, 다음 정보와 함께 반환됩니다:
   - 의심 점수 (0-1 사이 값)
   - 총 로그 수
   - 의심스러운 로그 수
   - 의심스러운 로그 항목과 탐지된 공격 패턴
   - 탐지된 공격 유형 요약

## 공격 탐지 방법

시스템은 다음과 같은 주요 공격 유형을 탐지합니다:
- SQL 인젝션
- XSS (크로스 사이트 스크립팅)
- 디렉토리 탐색
- 명령어 삽입
- 파일 포함 취약점
- 웹 취약점 스캐너 활동

각 공격 유형에 대해 최적화된 검사 전략을 적용하여 효율적인 탐지를 수행합니다.

## 추가 정보: 함수 사용 현황

현재 구현에서는 `isLogSuspicious()` 함수가 직접 사용되지 않고 있으며, 대신 `isLogSuspiciousFullScan()` 함수가 사용되고 있습니다:

1. **isLogSuspicious() (미사용)**
   - 공격 유형별로 최적화된 검사 전략을 사용합니다.
   - URL 경로, 쿼리 파라미터, 헤더, 본문 등 로그의 특정 부분만 선택적으로 검사합니다.
   - 공격 유형(SQL 인젝션, XSS, 디렉토리 탐색 등)에 따라 다른 검사 로직을 적용합니다.
   - 성능 최적화를 위해 설계되었으나 현재 활용되지 않습니다.

2. **isLogSuspiciousFullScan() (현재 사용 중)**
   - 모든 정규식 패턴을 로그 전체에 직접 적용하는 단순한 방식입니다.
   - 공격 유형을 구분하지 않고 모든 패턴을 모든 로그에 적용합니다.
   - `analyzeLogsForAttacks()` 함수 내에서 이 함수를 호출하여 로그 분석을 수행합니다.
   - 포괄적인 검사가 가능하지만 상대적으로 최적화되지 않은 접근 방식입니다.

향후 성능 개선이 필요한 경우, 최적화된 `isLogSuspicious()` 함수를 활용하도록 코드를 변경하는 것을 고려할 수 있습니다.
그러나 과소 탐지의 문제로 현재는 후자를 사용합니다.
